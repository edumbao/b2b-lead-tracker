Private workflow

Use GitHub for the tool, not the data.

  1. Upload this public app file to GitHub Pages.
  2. Keep your real CSV in a private local or cloud folder.
  3. Import the private CSV only inside the running web app.
  4. Export updates and save the file outside your GitHub folder.
  5. Clear local data after each session if the device is shared.

Safety checklist

Before publishing, check the repo.

  • Only upload index.html, README, privacy docs, and fake sample CSV.
  • Never upload real lead CSVs, exports, or client spreadsheets.
  • Use .gitignore to block CSV and spreadsheet files by default.
  • Think: GitHub is the public interface. Your computer is the private database.
